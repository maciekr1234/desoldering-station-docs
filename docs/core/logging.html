Logging System

Debug and diagnostic logging with severity levels.

Overviewâ€‹

The logging system provides macros for outputting debug messages with different severity levels. It wraps the Pico SDK logging with consistent formatting and optional output redirection.

Featuresâ€‹

  • 5 severity levels (Error, Warning, Info, Debug, Verbose)
  • Tag-based categorization
  • Compile-time level filtering
  • Custom output redirection
  • Minimal overhead when disabled

Severity Levelsâ€‹

LevelMacroUse Case
ErrorPICO_LOGECritical failures, requires attention
WarningPICO_LOGWUnexpected but recoverable situations
InfoPICO_LOGINormal operational messages
DebugPICO_LOGDDevelopment debugging info
VerbosePICO_LOGVDetailed trace information

API Referenceâ€‹

Logging Macrosâ€‹

#include "core/pico_log.h"

// Log with tag and format string
PICO_LOGE(tag, format, ...)  // Error
PICO_LOGW(tag, format, ...)  // Warning
PICO_LOGI(tag, format, ...)  // Info
PICO_LOGD(tag, format, ...)  // Debug
PICO_LOGV(tag, format, ...)  // Verbose

Configurationâ€‹

// Set minimum log level (compile-time)
#define PICO_LOG_LEVEL PICO_LOG_LEVEL_DEBUG

// Set output function (runtime)
void log_set_vprintf(int (*func)(const char*, va_list));

Example Usageâ€‹

Basic Loggingâ€‹

#include "core/pico_log.h"

static const char* TAG = "heater";

void heater_init() {
    PICO_LOGI(TAG, "Initializing heater controller");
    
    if (!setup_pwm()) {
        PICO_LOGE(TAG, "PWM setup failed on pin %d", CONFIG_HEATER_PIN);
        return;
    }
    
    PICO_LOGD(TAG, "PWM configured: freq=%dHz", CONFIG_HEATER_PWM_FREQ);
}

void heater_update(float temp) {
    PICO_LOGV(TAG, "Current temp: %.1fÂ°C", temp);
    
    if (temp > MAX_SAFE_TEMP) {
        PICO_LOGW(TAG, "Temperature %.1f exceeds safe limit!", temp);
    }
}

Conditional Loggingâ€‹

// Only compiled in debug builds
#ifdef DEBUG
    PICO_LOGD(TAG, "Debug info: state=%d, power=%d%%", state, power);
#endif

// Level filtering at compile time
#if PICO_LOG_LEVEL >= PICO_LOG_LEVEL_VERBOSE
    PICO_LOGV(TAG, "Detailed trace...");
#endif

Log Levels Configurationâ€‹

Compile-Time Filteringâ€‹

In platformio.ini:

build_flags = 
    -DPICO_LOG_LEVEL=PICO_LOG_LEVEL_INFO  ; Only INFO and above
DefineValueIncludes
PICO_LOG_LEVEL_NONE0Nothing
PICO_LOG_LEVEL_ERROR1Errors only
PICO_LOG_LEVEL_WARN2+ Warnings
PICO_LOG_LEVEL_INFO3+ Info
PICO_LOG_LEVEL_DEBUG4+ Debug
PICO_LOG_LEVEL_VERBOSE5Everything

Runtime Output Redirectionâ€‹

// Redirect to custom output (e.g., UART, display)
int my_log_output(const char* fmt, va_list args) {
    char buf[256];
    int len = vsnprintf(buf, sizeof(buf), fmt, args);
    uart_write(buf, len);
    return len;
}

void setup() {
    log_set_vprintf(my_log_output);
}

Best Practicesâ€‹

Tag Namingâ€‹

Use short, consistent tags:

static const char* TAG = "heater";   // âœ… Good

Message Guidelinesâ€‹

// âœ… Good: Actionable, includes context
PICO_LOGE(TAG, "Init failed: PWM slice %d unavailable", slice);

// âŒ Bad: Vague, no context
PICO_LOGE(TAG, "Error!");

// âœ… Good: Includes relevant values
PICO_LOGW(TAG, "Temp %.1fÂ°C exceeds limit %dÂ°C", temp, limit);

// âŒ Bad: Missing units/context

Performanceâ€‹

// âœ… Good: Avoid expensive operations if level disabled
if (PICO_LOG_LEVEL >= PICO_LOG_LEVEL_DEBUG) {
    float avg = compute_expensive_average();
    PICO_LOGD(TAG, "Average: %.2f", avg);
}
