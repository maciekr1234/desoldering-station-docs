Motion Sensor Driver

Pulse-width based motion detection for activity tracking.

Overviewâ€‹

The Motion Sensor driver detects valid pulses from a shake/motion sensor connected to a GPIO pin. It uses interrupt-driven edge detection to measure pulse width and filters out noise by requiring a minimum pulse duration. Used to detect handle activity for standby/sleep timeout management.

Featuresâ€‹

  • Interrupt-driven pulse detection
  • Configurable minimum pulse width filter
  • Callback on valid pulse detection
  • Time since last valid pulse tracking
  • Pulse width measurement

API Referenceâ€‹

Class Definitionâ€‹

class MotionSensor {
public:
    // Constructor
    // pin: GPIO pin connected to motion sensor
    // filter_us: Minimum pulse width in microseconds (default: 200ms = 200000us)
    MotionSensor(pin_size_t pin, uint32_t filter_us = 200000);
    ~MotionSensor();
    
    // Lifecycle
    void begin();
    void end();
    void update();  // Call periodically to process pulses
    
    // Callback for valid pulse detection
    void setCallback(std::function<void()> callback);
    
    // Filter configuration
    void setFilterTime(uint32_t filter_us);
    uint32_t getFilterTime() const;
    uint32_t getFilterTimeMs() const;
    
    // Pulse information
    uint32_t getLastPulseWidth() const;      // Last pulse width in microseconds
    uint32_t getLastPulseWidthMs() const;    // Last pulse width in milliseconds
    
    // Timing
    uint32_t getLastValidPulseTime() const;      // Timestamp of last valid pulse (us)
    uint32_t getLastValidPulseTimeMs() const;    // Timestamp in milliseconds
    uint32_t getTimeSinceLastValidPulseMs() const;  // Time since last valid pulse
};

Pulse Filteringâ€‹

The sensor filters out noise by requiring pulses to exceed a minimum duration:

Filter SettingValueUse Case
High sensitivity50msDetect small movements
Default200msNormal shake detection
Low sensitivity500msOnly deliberate shakes
// Set filter to 100ms (100000 microseconds)
motion.setFilterTime(100000);

Integration with Heaterâ€‹

Example Usageâ€‹

#include "drivers/motion_sensor.h"

// Motion sensor on GPIO 24, 200ms minimum pulse
MotionSensor motion(24, 200000);

void setup() {
    motion.begin();
    
    // Set callback for valid pulse detection
    motion.setCallback([]() {
        PICO_LOGI("MOTION", "Activity detected!");
        heater.resetActivityTimer();
    });
}

void motion_task() {
    while (true) {
        motion.update();
        
        // Check idle time for standby/sleep logic
        uint32_t idle_ms = motion.getTimeSinceLastValidPulseMs();
        
        if (idle_ms > SLEEP_TIMEOUT_MS) {
            heater.enterSleep();
        } else if (idle_ms > STANDBY_TIMEOUT_MS) {
            heater.enterStandby();
        }
        
        vTaskDelay(pdMS_TO_TICKS(100));
    }
}

Configuration in Settingsâ€‹

The motion sensor settings are stored in heater_settings_t.shake_sensor:

struct {
    bool     enabled;  // Enable/disable shake sensor
    bool     invert;   // Invert logic level (active low)
    uint32_t pulse_ms; // Minimum pulse duration in milliseconds
} shake_sensor;
