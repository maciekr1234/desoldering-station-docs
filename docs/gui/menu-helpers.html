Menu Helpers

Helper functions for creating settings menu items.

Overviewâ€‹

The menu_helpers module provides convenience wrappers around lv_settings_menu functions for common menu item patterns used in settings screens. These helpers reduce boilerplate code when building settings pages.

Usageâ€‹

Include the header in your settings screen implementation:

#include "gui/extra/menu_helpers.h"

API Referenceâ€‹

Page Creationâ€‹

create_pageâ€‹

Create a settings page with an optional section container.

std::pair<lv_obj_t*, lv_obj_t*> create_page(lv_obj_t* menu, 
                                             const char* title, 
                                             bool with_section = true);

Parameters:

  • menu â€” Parent settings menu object
  • title â€” Title displayed in the page header
  • with_section â€” Whether to create a section within the page (default: true)

Returns: A std::pair containing:

  • first â€” Pointer to the created page
  • second â€” Pointer to the created section (or nullptr if with_section is false)

Example:

auto [page, section] = create_page(menu, "Heater Settings");
// Add items to section...
return page;

Item Creationâ€‹

create_numericâ€‹

Create a numeric input item with configurable range, step, and unit.

lv_obj_t* create_numeric(lv_obj_t*     parent,
                         const char*   title,
                         int32_t       value,
                         lv_event_cb_t event_cb = nullptr,
                         int32_t       min      = INT32_MIN,
                         int32_t       max      = INT32_MAX,
                         int32_t       step     = 1,
                         const char*   unit     = nullptr);

Parameters:

  • parent â€” Section to add the item to
  • title â€” Label displayed for the item
  • value â€” Initial integer value
  • event_cb â€” Optional callback for LV_EVENT_VALUE_CHANGED
  • min / max â€” Value range limits
  • step â€” Increment/decrement step
  • unit â€” Unit suffix (e.g., " ms", " %")

create_numeric_floatâ€‹

Create a floating-point numeric input with decimal places.

lv_obj_t* create_numeric_float(lv_obj_t*     parent,
                               const char*   title,
                               float         value,
                               float         min,
                               float         max,
                               uint8_t       decimal_places = 2,
                               uint32_t      step           = 1,
                               const char*   unit           = nullptr,
                               lv_event_cb_t event_cb       = nullptr);

Parameters:

  • decimal_places â€” Number of decimal digits to display (e.g., 2 for 0.00)
  • step â€” Step in smallest units (e.g., 1 with 2 decimal places = 0.01)

Example:

create_numeric_float(section, "Kp", 2.5f, 0.0f, 100.0f, 2, 1);
// Displays: "Kp: 2.50" with Â±0.01 step

create_numeric_temperatureâ€‹

Create a temperature input with preset range from config.h.

lv_obj_t* create_numeric_temperature(lv_obj_t*     parent,
                                     const char*   title,
                                     int32_t       value,
                                     uint32_t      step     = 5,
                                     lv_event_cb_t event_cb = nullptr);

Uses CONFIG_MIN_SET_TEMP and CONFIG_MAX_SET_TEMP for range limits, and appends " Â°C" as the unit suffix.

Example:

create_numeric_temperature(section, "Target Temperature", 350, 10);
// Range: 50-450Â°C, step: 10, unit: " Â°C"

create_switchâ€‹

Create a boolean toggle switch.

lv_obj_t* create_switch(lv_obj_t*     parent,
                        const char*   title,
                        bool          state,
                        uint8_t       mode     = 0,
                        lv_event_cb_t event_cb = nullptr);

Parameters:

  • state â€” Initial switch state (true = on)
  • mode â€” Display mode:
    • 0 â€” "Enabled" / "Disabled"
    • 1 â€” "True" / "False"
    • 2 â€” "On" / "Off"

create_buttonâ€‹

Create a clickable action button.

lv_obj_t* create_button(lv_obj_t* parent, 
                        const char* title, 
                        lv_event_cb_t event_cb = nullptr);

The callback is triggered on LV_EVENT_CLICKED.

Example:

create_button(section, "Save Settings", [](lv_event_t* e) {
    save_global_settings();
});

create_subpageâ€‹

Create a navigation item that opens another page when clicked.

lv_obj_t* create_subpage(lv_obj_t* parent, 
                         const char* title, 
                         lv_settings_menu_create_page_t page_create_cb);

Parameters:

  • page_create_cb â€” Function that creates and returns the subpage
Memory Management

Subpages are created on-demand when clicked and deleted when navigating back to free memory immediately. This ensures RAM is freed as soon as you exit a subpage.

Example:

lv_obj_t* create_advanced_page(lv_obj_t* menu) {
    auto [page, section] = create_page(menu, "Advanced");
    // Add items...
    return page;
}

// In parent page:
create_subpage(section, "Advanced Settings", create_advanced_page);

create_frequency_boundâ€‹

Create a frequency input item bound to a uint32_t Hz value with automatic scale selection.

enum class FrequencyScale {
    Auto,  // Automatically select best scale based on value
    Hz,    // Display in Hertz (divisor = 1)
    kHz,   // Display in kilohertz (divisor = 1,000)
    MHz,   // Display in megahertz (divisor = 1,000,000)
    GHz,   // Display in gigahertz (divisor = 1,000,000,000)
};

lv_obj_t* create_frequency_bound(lv_obj_t*      parent,
                                 const char*    title,
                                 uint32_t*      value,
                                 uint32_t       min_hz,
                                 uint32_t       max_hz,
                                 uint32_t       step_hz = 1000,
                                 FrequencyScale scale   = FrequencyScale::Auto);

Parameters:

  • value â€” Pointer to uint32_t variable storing Hz value
  • min_hz / max_hz â€” Range in Hz
  • step_hz â€” Step increment in Hz
  • scale â€” Display scale (Auto selects best unit based on max value)

Behavior:

  • The bound variable always stores Hz internally
  • The UI displays and edits in the selected scale (kHz, MHz, etc.)
  • With FrequencyScale::Auto, the largest unit where max â‰¥ 1 is chosen

Example:

// 1-100 kHz range, 1kHz step â†’ displays as "1-100 kHz"
create_frequency_bound(section, "PWM Frequency", &gSettings.pump.frequency_hz,
                       1000, 100000, 1000);

Complete Exampleâ€‹

#include "gui/extra/menu_helpers.h"
#include "settings.h"

static lv_obj_t* create_heater_page(lv_obj_t* menu) {
    auto [page, section] = create_page(menu, "Heater");

    create_numeric_temperature(section, "Default Temperature", 
                               gSettings.heater.default_temperature, 10);
    
    create_numeric_temperature(section, "Standby Temperature", 
                               gSettings.heater.standby_temperature, 10);
    
    create_switch(section, "Auto-Standby", gSettings.heater.auto_standby, 0);

    return page;
}
