GUI Overview

LVGL-based graphical user interface for the desoldering station.

Overviewâ€‹

The GUI system uses LVGL (Light and Versatile Graphics Library) to render a modern, responsive interface on the station's display. It follows an observer pattern for reactive UI updates.

Featuresâ€‹

  • 60 FPS rendering
  • Dark/light theme support
  • Encoder navigation (no touch required)
  • Reactive data binding with lv_subject
  • Custom widgets for settings

Data Bindingâ€‹

The UI uses LVGL's observer pattern for reactive updates:

Subject Typesâ€‹

// In gui/global.h
extern lv_subject_t subject_current_temp;    // int32_t
extern lv_subject_t subject_target_temp;     // int32_t
extern lv_subject_t subject_heater_state;    // int32_t (enum)
extern lv_subject_t subject_power_percent;   // int32_t

Creating Bindingsâ€‹

// Create a label that auto-updates with temperature
lv_obj_t* temp_label = lv_label_create(parent);

// Bind to subject with format string
lv_label_bind_text(temp_label, &subject_current_temp, "%dÂ°C");

// Or use custom observer
lv_subject_add_observer(&subject_heater_state, state_changed_cb, label);

Task Integrationâ€‹

LVGL runs in a dedicated FreeRTOS task:

void lvgl_task(void* param) {
    while (true) {
        lv_timer_handler();
        vTaskDelay(pdMS_TO_TICKS(5));  // 5ms = 200Hz
    }
}
warning

LVGL is not thread-safe. Always use lv_lock()/lv_unlock() when accessing from other tasks:

void update_from_other_task() {
    lv_lock();
    lv_label_set_text(label, "Updated");
    lv_unlock();
}
