PID Auto-Tuning

Automatic PID parameter identification using the relay feedback method.

Overviewâ€‹

When to Useâ€‹

  • After replacing the heating element
  • If temperature control is sluggish or has excessive overshoot
  • When changing to a different tip type
  • As a starting point before manual fine-tuning

2. Runningâ€‹

The tuner alternates between full power (heating) and zero power (cooling) around the target temperature:

  1. Initial heating â€” Heats to target temperature

How It Worksâ€‹

Relay Feedback Methodâ€‹

The relay test creates sustained oscillations by:

  1. Applying 100% power when temperature < (target - hysteresis/2)
  2. Applying 0% power when temperature > (target + hysteresis/2)

This produces a limit cycle with measurable period and amplitude.

Thermal Lag Identificationâ€‹

The tuner also measures the delay between power changes and temperature response. This value is stored in gSettings.heater.thermal_lag_ms and can be used for feedforward compensation.

  1. Test the response â€” Set a target temperature and observe the heating curve

  3. Enable Soft Landing â€” Reduces overshoot near setpoint (see Heater Controller)

API Referenceâ€‹

#include "core/auto_tuner.h"

// Start with default config
auto_tune_start(nullptr);

// Or customize
auto_tune_config_t config = auto_tune_default_config();
config.target_temperature = 350;
config.hysteresis = 3.0f;
auto_tune_start(&config);

// Check progress
auto_tune_progress_t progress;
auto_tune_get_progress(&progress);

// Get results when complete
if (auto_tune_get_state() == AUTO_TUNE_COMPLETE) {
    auto_tune_result_t result;
    auto_tune_get_result(&result);
    // Apply result.pid_zn or result.pid_conservative
}
